# Adapter Contract

> Every Layer 1 source adapter conforms to this contract. New sources plug in by extending the `Adapter` ABC and registering with the dispatcher. Reading + following this doc is a prerequisite for the `Stage 2.x` sub-agent briefs.

## The ABC

```python
# ehi_atlas/adapters/base.py

from abc import ABC, abstractmethod
from pathlib import Path
from typing import Iterator
from pydantic import BaseModel


class SourceMetadata(BaseModel):
    """Per-bronze-record metadata, written alongside the artifact."""
    source: str                       # canonical source name (e.g. "synthea", "epic-ehi")
    patient_id: str                   # the patient this record is for
    fetched_at: str                   # ISO 8601 UTC
    document_type: str | None = None  # e.g. "lab-report", "ccda", "fhir-bundle"
    license: str                      # license string for redistribution
    consent: str                      # "open" | "personal" | "constructed" | "claim-only"
    sha256: str                       # content hash for reproducibility


class Adapter(ABC):
    """Layer 1 source adapter. Reads from _sources/<name>/raw/ and writes to bronze/<name>/<patient>/."""

    name: str  # canonical source name; matches the _sources/ subdirectory

    def __init__(self, source_root: Path, bronze_root: Path):
        self.source_root = source_root  # ehi-atlas/corpus/_sources/<name>/raw/
        self.bronze_root = bronze_root  # ehi-atlas/corpus/bronze/<name>/

    @abstractmethod
    def list_patients(self) -> list[str]:
        """Patient IDs available from this source."""

    @abstractmethod
    def ingest(self, patient_id: str) -> SourceMetadata:
        """Read raw, write canonical bronze record + metadata. Idempotent."""

    @abstractmethod
    def validate(self, patient_id: str) -> list[str]:
        """Return a list of validation errors; empty list means valid."""
```

## The contract in five points

**1. Idempotent.** Running `ingest()` twice on the same patient produces byte-identical output. No timestamps in payloads, no random IDs.

**2. Read-only on `_sources/`.** Adapters never modify their source-root inputs. If the source format requires preprocessing, do it inside `bronze/` not `_sources/`.

**3. Bronze layout is canonical.** Every adapter writes to:
```
corpus/bronze/<source>/<patient>/
├── metadata.json          # SourceMetadata as JSON
├── data.<ext>             # the source's primary representation
├── attachments/           # optional: extracted attachment binaries
│   └── <id>.<ext>
└── pages/                 # optional: PDF rasterizations + bbox text (if applicable)
    ├── 001.png
    └── 001.text.json
```

**4. Metadata is mandatory.** Every bronze record must have a `metadata.json` conforming to `SourceMetadata`. The `sha256` field is computed over `data.<ext>` and is the reproducibility anchor.

**5. Validation is structural.** `validate()` checks the bronze record's shape, not its semantic correctness. ("Did the JSON parse?" not "Are these clinically valid medications?")

## Registration

Adapters are registered in `ehi_atlas/adapters/__init__.py`:

```python
from .synthea import SyntheaAdapter
from .epic_ehi import EpicEhiAdapter
# ...

REGISTRY: dict[str, type[Adapter]] = {
    "synthea": SyntheaAdapter,
    "epic-ehi": EpicEhiAdapter,
    "ccda": CCDAAdapter,
    "blue-button": BlueButtonAdapter,
    "lab-pdf": LabPDFAdapter,
}
```

The CLI (`ehi-atlas ingest --source synthea`) looks up the registry, instantiates the adapter, and calls `ingest()` for the requested patient(s).

## Tests

Every adapter has a test file at `tests/adapters/test_<source>.py` with **at least three test inputs** covering:

1. A typical input (the showcase patient or a representative sample)
2. An edge case (empty section, missing required field, unusual encoding)
3. A failure case (malformed input → expected error)

Test fixtures live in `tests/fixtures/<source>/` and are committed to git when small (< 100KB) or generated by the test setup when large.

## Privacy gate

For sources containing real personal data:

- The adapter must read from `_sources/<source>/raw-redacted/` not `_sources/<source>/raw/`
- The adapter's `SourceMetadata.consent` field is `"personal"` and the bronze output is also gitignored (configured via `.gitignore`)
- The adapter's docstring states the privacy posture explicitly

## Adding a new source

Sub-agent recipe (the prompt template for adding a new adapter):

```
GOAL: Add a Layer 1 adapter for <new-source>.

READ FIRST:
- ehi-atlas/docs/ADAPTER-CONTRACT.md (this doc)
- ehi-atlas/ehi_atlas/adapters/synthea.py (canonical example, simplest case)
- ehi-atlas/corpus/_sources/<new-source>/README.md (the source's own readme)

DO:
- Implement <new-source>Adapter in ehi-atlas/ehi_atlas/adapters/<new-source>.py
- Register in ehi-atlas/ehi_atlas/adapters/__init__.py
- Write tests/adapters/test_<new-source>.py with ≥3 test inputs
- Update BUILD-TRACKER.md row 2.X to done in your final message (propose only)

DON'T:
- Modify _sources/ or bronze/ contents outside the test fixtures
- Add new dependencies without flagging in your final message
- Touch other adapters

OUTPUT PATH: ehi-atlas/ehi_atlas/adapters/<new-source>.py + test file
RETURN: punch-line summary, any architecture surprises, proposed tracker update.
```

## What "done" looks like for an adapter

- [ ] `ingest()` produces a valid bronze record for the showcase patient
- [ ] `validate()` returns `[]` on the showcase patient
- [ ] `metadata.json` has all required `SourceMetadata` fields
- [ ] Tests pass with ≥3 inputs
- [ ] Re-running `ingest()` is byte-identical (idempotent)
- [ ] Privacy gate respected if applicable
- [ ] Adapter registered in `REGISTRY`

## Using BundleValidator (Layer 2 silver-gate)

After a standardize step converts a bronze record into a FHIR R4 Bundle dict,
pass it through `BundleValidator` before writing to `corpus/silver/`. The
validator is the schema gate at the bronze → silver boundary. An empty error
list means the bundle is safe for Layer 3 (harmonize) to consume.

### When to call it

- Inside any standardize function that produces a silver Bundle (e.g. a Synthea
  passthrough, a CCDA-to-FHIR conversion, an Epic TSV mapping).
- From the `ehi-atlas validate --bundle <path>` CLI for ad-hoc inspection of
  any silver file on disk.

### Import and basic usage

```python
from ehi_atlas.standardize.validators import BundleValidator, validate_bundle

# Option A — convenience function (non-strict by default)
errors = validate_bundle(bundle_dict)
if errors:
    raise ValueError(f"Silver bundle failed validation: {errors}")

# Option B — class (use strict=True to promote unknown profiles to errors)
validator = BundleValidator(strict=True)
messages = validator.validate(bundle_dict)

errors   = [m for m in messages if not m.startswith("warning:")]
warnings = [m for m in messages if m.startswith("warning:")]
```

### What the validator checks

| Layer | Check | On failure |
|---|---|---|
| 1. Structural | Bundle parses as valid FHIR R4 via `fhir.resources`; `type` field present | hard error; downstream checks skipped |
| 2. Profile | Every `resource.meta.profile[]` URL is in `KNOWN_PROFILES` (USCDI + CARIN BB) | warning (non-strict) or error (strict) |
| 3. Provenance tags | Every resource `meta.tag[]` covers both required systems: `source-tag` + `lifecycle` | hard error |
| 3b. Extraction Extensions | Resources with `lifecycle=extracted` must have `extraction-model`, `extraction-confidence`, and `source-attachment` Extensions on `meta` | hard error |

### CLI usage

```bash
# Validate a silver file on disk
ehi-atlas validate --bundle corpus/silver/synthea/rhett759/bundle.json

# Strict mode: unknown profiles are errors
ehi-atlas validate --bundle corpus/silver/ccda/rhett759/bundle.json --strict
```

Exits 0 on success; exits 1 if any hard errors are found. Warnings are printed
but do not block the exit.

### Adding new profiles

Add the StructureDefinition URL to `BundleValidator.KNOWN_PROFILES` in
`ehi_atlas/standardize/validators.py`. The set is intentional: only profiles
we have explicitly reviewed belong there. Unknown profiles from vendor-specific
IGs trigger warnings so the pipeline doesn't silently accept arbitrary claims.
