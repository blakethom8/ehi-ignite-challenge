## What this source is

A synthesized Quest-style lab report PDF for the showcase patient, deliberately constructed to:

- Contain a creatinine value (LOINC `2160-0`) at `1.4 mg/dL`, dated `2025-09-12`
- The same creatinine value also appears in the showcase patient's FHIR Observations (slightly different format)
- Layer 2-B vision extraction lifts the value into a FHIR Observation
- Layer 3 then merges this PDF-derived Observation with the matching FHIR Observation via (LOINC, effectiveDateTime, value, unit) hash
- Sources panel walks the merge Provenance back to the PDF and highlights page 2, bbox `72,574,540,590`

This is **Artifact 5** in the showcase patient construction recipe.

## How to acquire

The PDF is generated by a script committed to `_sources/synthesized-lab-pdf/generator.py`:

```bash
cd ehi-atlas/corpus/_sources/synthesized-lab-pdf/
uv run python generator.py
# → produces raw/lab-report-2025-09-12-quest.pdf
# NOTE: requires reportlab in pyproject.toml — add it if not present:
#   uv add reportlab
```

The script uses `reportlab` to lay out a multi-page lab report with:
- Page 1: header (patient demographics, ordering provider, lab info), test summary table
- Page 2: detailed lab values (the creatinine row at the documented bbox)
- Page 3: reference ranges, flags, footer

## Reproduction recipe

The script is deterministic. Re-running produces a byte-identical PDF (font + content fixed; no timestamps in the output).

## Privacy gate

Not applicable — fully synthetic, generator script committed.

## Why we constructed this

Real lab PDFs from Cedars or Quest are personal data. A synthetic equivalent lets us:

- Demo Layer 2-B vision extraction publicly
- Have deterministic, reproducible test inputs
- Plant the cross-source merge artifact (Artifact 5) without privacy concerns
- Iterate on extraction prompts in notebooks without leaking PHI
