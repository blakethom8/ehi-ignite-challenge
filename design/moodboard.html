Agent Panel Direction

Each module should have a dedicated agent with scoped tools, prompt, and citations. The UI should make the agent role obvious.

Patient Record Agent: answer neutral chart questions with source citations.
Pre-Op Agent: reason about holds, anesthesia handoff, and clearance gaps.
Trials Agent: search registries, parse criteria, rank candidate studies.
Access Agent: find pricing, support programs, and eligibility requirements.

06 Rules

Guardrails for the redesign pass.

These are implementation rules to keep the UI from drifting back into generic cards and gray dashboards.

Do

  • Use module overview pages for orientation and patient pages for data.
  • Keep patient-specific pages compact, row-based, and evidence-forward.
  • Show source type, date, and confidence when making assertions.
  • Use color only for status, module identity, and actions.
  • Prefer structured panels over repeated generic cards.

Avoid

  • Oversized decorative cards inside clinical workspaces.
  • Gray-on-gray dashboards with no clinical hierarchy.
  • Marketing-style hero sections on operational pages.
  • Mixing module instructions and patient findings on one page.
  • Using the same agent UI without clarifying the module role.
